# Тестовое задание (Data engineering)

## Описание
> Докер контейнер с самозаполняемой базой данных пользовательских логов, который по запросу делает аггрегацию данных

> В данном проекте реализованы файлы docker-compose и Makefile, ниже приведены инструкции пользования
## Установка
> [!IMPORTANT]
> 1. Заменить файл .env.example на файл .env со своими данными
> 2. В папке с проектом запустить:
```bash
make init
```
поднимется контейнер с python, там установятся нужные библиотеки, поднимутся контейнеры с БД и PGAdmin4 для наглядного просмотра, сгенерируется датасет, содержащий логи за 30 дней.

> [!NOTE]
> PGAdmin4 откроется на локалхосте http://localhost:8080/


> [!TIP]
> Если нужно очистить базу данных и заполнить ее новыми данными:
```bash
make reset
```

> [!TIP]
> Остановить все:
```bash
make down
```
## Использование (пример)
> После установки и поднятия контейнеров, в консоли будет предложен вариант ввода команды для аггрегации данных, будет приниматься любой валидируемый тип даты, даже 13march2026 главное писать без пробелов:)

> [!NOTE]
> make aggregate START="" END=""
так же можно написать просто make aggregate чтобы получить аггрегацию за текущий месяц ,с 1 числа

### 2.  Исследование активности пользователей (на основе данных VL.ru)

Для реализации реалистичного генератора логов было проведено мини-исследование активности жителей Владивостока на городском портале VL.ru (раздел "Опросы").

Методология:
+ С помощью инструментов разработчика (JS Console) были собраны даты публикации более 500 комментариев из последних 3 тем.
+ Данные были нормализованы по дням недели.
+ Рассчитаны коэффициенты относительно самого спокойного дня (среда = 1.0).

Полученные коэффициенты:
+ **Понедельник:** 1.7
+ **Вторник:** 2.2
+ **Среда:** 1.0 (базовый уровень)
+ **Четверг:** 4.8 (пик обсуждений)
+ **Пятница:** 3.7
+ **Суббота:** 2.2
+ **Воскресенье:** 2.6

Эти коэффициенты внедрены в `generator.py` для симуляции реального поведения аудитории. Конечно может показаться притянуто за уши, но мне кажется, что цифры не врут, да и мне хотелось получить какие-то статистические данные для более правдоподобной активности.
